litoralulromanesc.ro — desktop redesign

New desktop designs for litoralulromanesc.ro, built on the approved Szallas Group redesign (Redesign sandbox) — the same move as the Hotely.cz re-skin in the sandbox file: identical layout & components, LITRO brand skin, Romanian seaside content. Branding decisions follow the already-refreshed LITRO mobile redesign (navy rating badge, orange stars, red discount pill, navy price, FRIENDS credits).

▶ Două variante de homepage, paralele. A — clasic (hero + carusele). B — homepage-ul este direct listingul de oferte, cu filtre și carduri de deal (model travelist.pl/hotele), potrivit pentru inventar propriu unde oferta este produsul. Comută între ele cu switch-ul din dreapta ecranului.

▶ Acesta este un prototip interactiv, nu un mockup. Deschide oricare ecran și clic: caută o stațiune, schimbă datele în calendar (prețurile se recalculează pe zi), filtrează rezultatele, deschide galeria, alege o cameră, aplică codul promo LITORAL10, bifează condițiile și finalizează rezervarea — starea se transmite de la un ecran la altul.
This is a working prototype, not a mockup. Search, pick dates in the calendar (prices recompute per night), filter, open the gallery, select a room, apply promo code LITORAL10, tick the consent and complete the booking — state carries across screens.

Handoff — back to Figma & to dev

To Figma (ready-made): ⬇ litro-desktop-redesign.fig — a native Figma file with all five screens as editable frames (layers, texts, photos, exact sizes; single-line texts are auto-width). Drag it onto the Figma home screen / “Import” — done. Fonts used: Outfit + DM Sans (both in Figma’s Google Fonts catalog).

Alternative route: the html.to.design plugin (already used by the team for “LITRO Master Pages”) — capture these live pages: home.html, listing.html, hotel.html, checkout.html, thankyou.html from this site’s URL, or via its Chrome extension.

To dev: proto.js + proto.css hold all prototype behaviour (state, calendar, filters, gallery, checkout logic) and can be thrown away at implementation time; litro.css is the token sheet (colors, radii, shadows, type as CSS variables — all values in px) and specs.html is the measurement spec (component-by-component). The HTML files themselves are reference implementations — paddings, gaps and font sizes can be lifted 1:1.
Scope note. All 5 templates of the priority flow are covered: listing → hotel details → checkout → thank you, plus homepage. Checkout follows the LITRO mobile redesign model (single page: details + payment, 30% advance option) skinned with the Szallas DS. Photos are placeholders taken from the sandbox assets; swap for real LITRO property photos in production.
